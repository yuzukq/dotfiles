# VRMA生成 — あなたがモーションデザイナー

推測でキーフレームを打たず、まずリファレンスの座標規約と解剖学メモを読み、実際の人間の動きを頭で再生してから数値化する。変換スクリプトは依存ゼロ (Node 20+ のみ)。theo などのサーバーは不要。

## 手順

1. **事実確認**(必ず最初に):
   - `~/.claude/skills/vrma-create/reference.md` を全文読む(座標規約・ボーン一覧・可動域・作例)
   - 出力先を決める: ユーザー指定があればそれ、なければカレントディレクトリの `<モーション名>.vrma`。同名ファイルが既に存在する場合は上書きせず、ユーザーに確認するか `-2` 等の別名にする
   - モーション名は英数字とハイフン/アンダースコアのみ (例: `happy-dance`, `bow_deep`)
2. **spec設計**: reference.md の規約に従い、動きをキーフレームJSONにする。
   - その動きを実際の人間がどうやるか(関節の順序・リズム・重心移動)を先に言語化してから数値化する
   - t=0 は腕を下ろした自然な姿勢 (leftUpperArm Z=-70 / rightUpperArm Z=+70)
   - 感情に合った expressions (happy / sad / blink 等) を入れる
   - spec を一時ファイルに書く (例: `/tmp/vrma-spec.json`)
3. **変換実行**:
   ```sh
   node ~/.claude/skills/vrma-create/build-vrma.mjs /tmp/vrma-spec.json <出力先>.vrma
   ```
   - 出力はJSON 1行: `{"ok":true,"out":...,"bytes":...,"durationS":...,"warnings":[...]}` / 失敗時 `{"ok":false,"error":...}`
4. **検証**:
   - `ok:true` かつファイルが生成されていること (`ls -la` でサイズ確認、数KB以上あること)
   - **warnings を必ず読む**。可動域クランプや削除の警告は「specの設計ミス」の教材。警告が出たら spec の該当箇所を修正して再生成する (最大2回まで。3回目も警告が残るなら、警告内容と共にユーザーへ報告して判断を仰ぐ)
   - `{"ok":false}` の場合はエラーメッセージに従って spec を修正して再実行
5. **報告**(最後に必ず):
   - 出力パス・ファイルサイズ・duration
   - 残っている warnings (あれば)
   - どんな演技として設計したか1〜2文 (例: 「0.3秒の溜め→2回のお辞儀→余韻0.5秒、relaxed 0.5維持」)
   - 再生方法の一言 (VRMA対応アプリ、three.js + @pixiv/three-vrm-animation、theo など)

## specフォーマット (詳細と作例は reference.md)

```json
{
  "name": "モーション名",
  "duration": 2.4,
  "loop": false,
  "tracks": { "ボーン名": [ { "t": 0, "r": [X度, Y度, Z度] } ] },
  "hips": [ { "t": 0, "p": [0, 0, 0] } ],
  "expressions": { "happy": [ { "t": 0, "w": 0.8 } ] }
}
```

## 禁止事項

- reference.md を読まずに spec を書くこと (座標規約の思い込みは左右反転・破綻ポーズの元)
- warnings を無視して完了報告すること (クランプされた spec は意図と違う動きになっている)
- `ok:true` とファイル存在を確認せずに完了報告すること
- 作例の角度・キー配置をそのままコピーすること (毎回その動きに合わせて設計する)
- npm install 等の依存追加 (スクリプトは依存ゼロで動く。動かないのは Node < 20 の場合のみ)
- 既存の .vrma ファイルを確認なしに上書きすること
